# Deploy Product Catalog Service to Azure VM (Single B1s)

This guide walks you through deploying your entire CQRS microservice stack to a single Azure VM using free tier.

## Prerequisites

### Windows Setup

- **Azure Account** (free tier eligible)
- **Azure CLI** installed and logged in
- **Docker Desktop** installed locally (includes Docker CLI)
- **PowerShell 5.1+** or **Windows Terminal** (recommended)
- **Git Bash** (optional, for native bash commands)

**Verify prerequisites in PowerShell:**
```powershell
az --version
docker --version
az login  # This will open browser for login
```

### Linux/macOS Setup

- Same prerequisites, but use Bash shell instead of PowerShell
- SSH keys auto-generated by Azure CLI

The script will:
1. Create resource group and Container Registry
2. Build and push Docker image
3. Provision B1s VM
4. Configure networking (NSG/firewall)
5. Install Docker on VM
6. Upload and start docker-compose
7. Verify deployment is healthy

⏱️ **Total time: 8-12 minutes**

## Option B: Manual Step-by-Step Deployment

### Windows PowerShell

#### Step 1: Set environment variables

```powershell
# Configuration
$RESOURCE_GROUP = "rg-product-catalog-dev"
$VM_NAME = "product-catalog-vm"
$LOCATION = "eastus"
$ACR_NAME = "acrproductcatalog$((Get-Random))"
$VM_IP = ""

# Secrets (CHANGE THESE!)
$POSTGRES_PASSWORD = "YourSecurePass123!@#"
$COUCHBASE_PASSWORD = "YourSecurePass123!@#"
$APP_JWT_SECRET = [Convert]::ToBase64String([System.Security.Cryptography.RNGCryptoServiceProvider]::new().GetBytes(48))
# Step 2: Create resource group (PowerShell)

```powershell
az group create `
  --name $RESOURCE_GROUP `
  --location $LOCATION
```

#### Step 3: Create Azure Container Registry (PowerShell)

```powershell
az acr create `
  --resource-group $RESOURCE_GROUP `
  --name $ACR_NAME `
  --sku Basic
```

#### Step 4: Build and push Docker image (PowerShell)

```powershell
# Login to registry
az acr login --name $ACR_NAME

# Build image
$ACR_LOGIN_SERVER = "$ACR_NAME.azurecr.io"
docker build `
  -t "$ACR_LOGIN_SERVER/product-catalog-service:latest" `
  .

# Push to registry
docker push "$ACR_LOGIN_SERVER/product-catalog-service:latest"
```

#### Step 5: Create VM (PowerShell)

```powershell
$VM_IP = (az vm create `
  --resource-group $RESOURCE_GROUP `
  --name $VM_NAME `
  --image UbuntuLTS `
  --size Standard_B1s `
  --generate-ssh-keys `
  --public-ip-address-allocation static `
  --query "publicIpAddress" `
  -o tsv)

Write-Host "VM created with IP: $VM_IP"
```

#### Step 6: Configure firewall (PowerShell)

```powershell
# Allow SSH
az network nsg rule create `
  --resource-group $RESOURCE_GROUP `
  --nsg-name "${VM_NAME}NSG" `
  --name "AllowSSH" `
  --priority 100 `
  --direction Inbound `
  --access Allow `
  --protocol Tcp `
  --destination-port-ranges 22

# Allow HTTP (app port 8080)
az network nsg rule create `
  --resource-group $RESOURCE_GROUP `
  --nsg-name "${VM_NAME}NSG" `
  --name "AllowHTTP" `
  --priority 200 `
  --direction Inbound `
  --access Allow `
  --protocol Tcp `
  --destination-port-ranges 8080
```

# S# Step 7: Install Docker on VM (PowerShell SSH)

```powershell
# Connect to VM using ssh
ssh azureuser@$VM_IP

# Once connected to VM, run these commands:
sudo apt-get update
sudo apt-get install -y `
  apt-transport-https `
  ca-certificates `
  curl `
  gnupg `
  lsb-release

# Add Docker repository
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg

# Install Docker
echo "deb [arch=amd64 signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

sudo apt-get update
sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

# Allow docker without sudo
sudo groupadd -f docker
sudo usermod -aG docker $USER
newgrp docker

# Exit SSH
exit
```

> **Note:** On Windows 11, you can use `ssh` directly in PowerShell. For Windows 10, install [Git for Windows](https://git-scm.com/download/win) to get `ssh` command.

#### Step 8: Upload docker-compose and environment file (PowerShell)

```powershell
# Create .env file locally
$env_content = @"
ACR_LOGIN_SERVER=$ACR_LOGIN_SERVER
IMAGE_TAG=latest
POSTGRES_USER=catalog_user
POSTGRES_PASSWORD=$POSTGRES_PASSWORD
COUCHBASE_USER=Administrator
COUCHBASE_PASSWORD=$COUCHBASE_PASSWORD
APP_JWT_SECRET=$APP_JWT_SECRET
SPRING_PROFILES_ACTIVE=prod
"@

# Save to file
$env_content | Out-File -FilePath "$env:TEMP\.env" -Encoding UTF8

# Upload files using scp (via Git Bash or native ssh)
scp "$env:TEMP\.env" "azureuser@${VM_IP}:/tmp/.env"
scp "docker-compose.yml" "azureuser@${VM_IP}:/tmp/docker-compose.yml"

Write-Host "Files uploaded successfully"
```

#### Step 9: Start services on VM (PowerShell SSH)

```powershell
# SSH to VM
ssh azureuser@$VM_IP

# On the VM, run:
cd /tmp
export $(cat .env | xargs)

# Login to ACR
az acr login --name $(echo $ACR_LOGIN_SERVER | cut -d'.' -f1)

# Start all services
docker compose up -d

# Wait for services to start
sleep 15

# Check status
docker compose ps
docker compose logs --tail=20

# Exit SSH
exit
```

#### Step 10: Verify deployment (PowerShell)

```powershell
# From your local machine
$response = Invoke-WebRequest -Uri "http://$VM_IP:8080/actuator/health" -UseBasicParsing
$response.Content

# Should return: {"status":"UP"}
```

## Testing the Deployment

### Run automated end-to-end tests

```bash
chmod +x scripts/test-e2e-azure.sh
./scripts/test-e2e-azure.sh "http://$VM_IP:8080"
```Testing (PowerShell)

```powershell
# 1. Access Swagger UI in browser
Start-Process "http://$VM_IP:8080/swagger-ui.html"

# 2. Create a product (Command)
$productData = @{
    name = "Laptop"
    description = "High-end gaming laptop"
    price = 1299.99
    stockQuantity = 10
} | ConvertTo-Json

Invoke-WebRequest -Uri "http://$VM_IP:8080/api/v1/products" `
  -Method POST `
  -Headers @{ "Content-Type" = "application/json" } `
  -Body $productData `
  -UseBasicParsing

# 3. List products (Query - from CouchBase)
Invoke-WebRequest -Uri "http://$VM_IP:8080/api/v1/products" `
  -UseBasicParsing | Select-Object -ExpandProperty Content

# 4. Get specific product
# Replace {product-id} with actual product ID from step 2
Invoke-WebRequest -Uri "http://$VM_IP:8080/api/v1/products/{product-id}" `
  -UseBasicParsing | Select-Object -ExpandProperty Content

# 5. Update stock
$stockData = @{
    quantity = 5
    reason = "sales"
} | ConvertTo-Json

Invoke-WebRequest -Uri "http://$VM_IP:8080/api/v1/products/{product-id}/stock" `
  -Method PUT `
  -Headers @{ "Content-Type" = "application/json" } `
  -Body $stockData `
  -UseBasicParsing
```

```bash
ssh -i ~/.ssh/id_rsa azureuser@$VM_IP
cd /tmp
docker compose restart
```

### Common issues

| Issue | Solution |
|-------|----------|
| **Connection refused** | Wait 2-3 min for services to start |
| **Out of memory** | VM is undersized; B1s has only 1GB RAM - may need Standard_B2s |
| **Kafka connection timeout** | Ensure Kafka service healthcheck passes: `docker compose logs kafka` |
| **Couchbase not initializing** | Wait 3-5 min for Couchbase setup; check: `docker compose logs couchbase` |
| **Application crashed** | Check logs: `docker compose logs product-catalog-service` |
